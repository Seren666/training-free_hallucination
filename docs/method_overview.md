# Method Overview

## Problem

Large vision-language models can describe objects that are not supported by the image. This repository studies object hallucination in open-ended caption generation, where the output is a free-form caption rather than a yes/no answer.

## Setting

OCV is a post-decoding, training-free mitigation strategy. It starts from an existing caption produced by regular decoding or by a stronger source-generation method such as FLB. It then verifies object commitments in the generated text and applies bounded edits only when the evidence indicates that a mention is likely unsupported.

## High-Level Method

The public method description uses three evidence families:

- **Uncertainty evidence**: low mention probability or weak token confidence can indicate unsupported commitments.
- **Visual-support evidence**: weak attention or alignment with image tokens can indicate that a generated object is not grounded.
- **Anchoring/context evidence**: local caption context can rescue valid mentions and reduce unsafe deletions.

The implementation is intentionally bounded. It does not regenerate a full caption. It scores candidate object mentions, keeps conservative gates for risky edit cases, and applies small text edits to existing captions.

## Inputs And Outputs

Input:

- image-caption pairs generated by a base LVLM
- object mention rows extracted from captions
- optional model-internal evidence from a verification forward pass

Output:

- edited captions
- aggregate hallucination metrics
- edit and preservation statistics

## What This Repo Implements

This public repository includes:

- lightweight score aggregation utilities in `src/hallucination_mitigation/scoring.py`
- bounded span-removal utilities in `src/hallucination_mitigation/post_decoding.py`
- aggregate result records under `results/public/`
- table-building and repository hygiene scripts under `scripts/`

Model-specific probe runners and raw output artifacts are not redistributed because they depend on local checkpoint/data layouts and produce large raw generations.

## What This Repo Does Not Claim

- It does not claim a paper acceptance.
- It does not claim state of the art.
- It does not redistribute model weights, datasets, or full generated captions.
- It does not treat POPE as the main benchmark for the final caption-side results; POPE is only a diagnostic VQA-side record in the broader project history.
